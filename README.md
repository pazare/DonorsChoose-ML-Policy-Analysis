# Summary of Policy Analysis
Across the United States, schools that face persistent funding challenges force teachers to personally finance classroom supplies or seek support through crowdfunding platforms like DonorsChoose. While DonorsChoose enables educators to request funding for materials ranging from basic classroom supplies to lab equipment and field trips, about one-third of posted projects go unfunded. Failure to reach these goals disproportionately affects students in under-resourced schools and limits their access to quality learning experiences. Our project aims to support DonorsChoose in selecting projects for review. Specifically, we built a predictive model to identify the 10% of newly posted projects most at risk of going unfunded without timely and personalized intervention. By focusing on the projects with the lowest predicted funding success, we aim to maximize the impact of human intervention.
Our analysis identified XGBoost as the most effective classification algorithm after evaluating several models using 5-fold stratified cross-validation (mean ROC AUC: 0.7663). Hyperparameter tuning via Grid Search further optimized the XGBoost model, achieving a best cross-validation ROC AUC of 0.7559. On the held-out test data, the final tuned model yielded an ROC AUC of 0.7572. To best align with the goal of identifying at-risk (unfunded) projects while managing intervention resources, we optimized the classification threshold using Youden's J statistic. This resulted in an optimal threshold of 0.68, at which the model achieves a Specificity of 0.68 (correctly identifying 68% of unfunded projects) and a Recall (Sensitivity) of 0.68 (identifying 68% of funded projects). This threshold provides a strong balance for targeting the bottom 10% of projects most likely to fail for review by the digital content expert.
In the future, it would be helpful to integrate our selected machine learning model with a database resource management system and a user interface for ad hoc analysis. This would result in a user-friendly decision support system (DSS). Adding intervention analysis would increase the efficiency of the intervention process and improve the operability of the model. However, our model did not include intervention analysis since we mainly based our recommended project list on invariable features (i.e., school information, project information).
